# TradingView Chart Selector Bot

Telegram-бот выбирает лучший график TradingView для анализа криптовалюты. Он проверяет популярные биржи, сравнивает пары USD и USDT, оценивает длину истории и качество часовых свечей.

## Порядок выбора

Главный критерий - максимальная история графика. Длина истории проверяется по реальным свечам конкретного TradingView-символа на старших таймфреймах: месячные, недельные и дневные свечи. Качество графика проверяется отдельно по часовым свечам TradingView. Биржевые metadata и склеенная история старых тикеров не используются для выбора лучшего графика. USD/USDT, чистота свечей и ликвидность используются как дополнительные критерии, но не должны перебивать большую разницу в истории.

## Источник свечей

Поиск кандидатов по биржам остается через ccxt, но история и качество графика считаются по свечам TradingView для конкретного символа вида `BINANCE:BTCUSDT`. Если TradingView временно не отдает свечи по символу, бот использует запасной источник биржи, добавляет предупреждение и снижает оценку такого кандидата. Это важно для ребрендов и миграций: биржевой API может отдавать склеенную историю старого тикера, а TradingView-график начинается позже. Поэтому выбор графика не должен опираться на биржевые OHLCV, если доступны TradingView-свечи.

## Что оценивается

- Самая длинная доступная история.
- Приоритет USDT для монет, которые активно торгуются после массового появления USDT.
- USD как запасной или более подходящий вариант для старых активов.
- Разрывы в часовых свечах.
- Плоские свечи.
- Нулевой объем.
- Подозрительные скачки цены.
- Средний объем как грубая оценка ликвидности.
- Наличие фьючерсного рынка на MEXC: если его нет, бот показывает предупреждение.

## Биржи

По умолчанию используются провайдеры через ccxt: Binance, Bybit, Coinbase, Kraken, Bitstamp, OKX, Bitfinex, KuCoin, MEXC, Gate.io, Bitget, Crypto.com, Gemini.

Результат отдается в формате TradingView, например BINANCE:BTCUSDT или COINBASE:BTCUSD.

## Запуск локально

1. Создай бота у BotFather и получи токен.
2. Скопируй .env.example в .env.
3. Заполни BOT_TOKEN.
4. Установи зависимости: pip install -r requirements.txt
5. Запусти бота: python -m app.main

## Ответ бота

В ответе бот показывает основной TradingView-символ, оценку качества, метрики качества, 2-3 альтернативы и строку `Фьючерсы MEXC: Да/Нет`. Если фьючерсов на MEXC нет, ответ содержит предупреждение.

## Команды

- /start - краткая подсказка.
- BTC - найти лучший график.
- /why BTC - показать выбранный график и причины.
- /compare BTC - показать топ до 10 вариантов.

## Важные замечания

TradingView не предоставляет официальный открытый API для массового поиска всех символов. Поэтому проект берет рыночные данные через API бирж с помощью ccxt, а затем формирует символы в формате TradingView. Это стабильнее, чем зависеть от внутренних неофициальных механизмов TradingView.

Исторические свечи у бирж часто ограничены последними 500-1000 свечами за один запрос. Поэтому бот делает отдельные запросы за историей по старшим таймфреймам: месячные, недельные и дневные свечи. Для каждого таймфрейма он берет пачку до 1000 OHLCV-свечей и выбирает самую раннюю реально полученную свечу именно этой пары. Качество проверяется отдельно по свежему окну часовых свечей. Это не грузит всю многолетнюю историю целиком, но позволяет корректно учитывать возраст графика. Следующий production-шаг - добавить полную пагинацию истории и ночное обновление кеша.
